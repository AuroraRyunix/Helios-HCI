Workload Options

Choose the best workload engine or Kubernetes distribution to deploy inside guest virtual machines on Helios HCI.

K3s (Lightweight Kubernetes)

Easiest to Deploy

Developed by Rancher, K3s is a highly lightweight, fully compliant Kubernetes distribution packaged as a single binary. It reduces memory overhead to under 50MB per agent, making it the perfect workload hypervisor for Helios HCI guest VMs.

# Deploy K3s inside a guest VM in under 30 seconds: curl -sfL https://get.k3s.io | sh -
  • Boots instantly using an embedded SQLite database
  • Packaged with local path storage & Flannel CNI
  • Extremely low memory footprint & resource usage
  • Single binary orchestrating all Kubernetes elements

RKE2 (Government-Grade K8s)

Highly Secure

Also known as Rancher Next-Generation Kubernetes, RKE2 inherits VMware and RHEL parity compliance profiles. It focuses entirely on security, government compliance, and strict validation of system components.

  • FIPS 140-2 compliance ready for secure datacenters
  • Automated CIS benchmark checks applied out-of-the-box
  • Runs components as rootless containerized processes
  • Perfect for regulated multi-tenant production zones

Tanzu Kubernetes Grid (TKG)

Enterprise Complex

VMware's flagship Kubernetes platform. While it provides enterprise-grade operations and native vSphere control plane parity, it carries high infrastructure complexity, requiring dedicated management clusters and control-plane controllers.

  • Deep integration with multi-tenant storage policies
  • High management overhead requiring a dedicated VM cluster
  • Complex setup involving Tanzu CLI & bootstrap controllers
  • Best suited for legacy enterprise app migrations
